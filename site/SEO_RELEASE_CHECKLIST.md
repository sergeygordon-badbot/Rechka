# SEO-публикация страницы «Речки»

## Файлы

- содержимое `site/rechka/` публикуется как самостоятельный продукт
  по адресу `https://ebsf.ru/rechka/`;
- sitemap этого направления публикуется как
  `https://ebsf.ru/rechka/sitemap.xml`;
- корневые файлы и главная страница `ebsf.ru` этим релизом не изменяются.

После публикации страница, sitemap, CSS и JavaScript направления должны
отвечать кодом HTTP 200.

## Google Search Console

1. Подтвердить доменный ресурс `ebsf.ru`, если он ещё не подтверждён.
2. Открыть проверку URL `https://ebsf.ru/rechka/`.
3. Запустить проверку опубликованной страницы и убедиться, что индексирование
   разрешено, canonical совпадает с адресом страницы, CSS и JavaScript доступны.
4. Нажать «Запросить индексирование».
5. Добавить файл `https://ebsf.ru/rechka/sitemap.xml` в раздел Sitemaps.
6. После обхода проверить запросы, показы, CTR и выбранный Google canonical.

## Яндекс Вебмастер

1. Подтвердить сайт `https://ebsf.ru/`, если он ещё не подтверждён.
2. Убедиться, что корневой robots.txt не запрещает обход `/rechka/`.
3. Добавить `https://ebsf.ru/rechka/sitemap.xml` в раздел «Файлы Sitemap».
4. Открыть «Переобход страниц» и отправить `https://ebsf.ru/rechka/`.
5. После обхода проверить статус страницы, выбранный canonical, заголовок
   и описание сниппета.

## Контроль после публикации

- страница открывается как с завершающим `/`, так и без него, но один вариант
  перенаправляет на canonical;
- HTTP-версия и `www` сохраняют путь и перенаправляют на
  `https://ebsf.ru/rechka/`;
- в исходном HTML есть один H1: «Не печатайте. Скажите.»;
- точная категория «программа для голосового ввода» естественно раскрыта
  в title, первом абзаце и описании приложения;
- ссылки на установщик, GitHub и политику конфиденциальности не возвращают ошибку;
- разметка `SoftwareApplication` проходит проверку Schema.org без
  синтаксических ошибок;
- подпись digital-команды не перетягивает внимание с продукта и не выглядит
  как рекламный баннер агентства.
